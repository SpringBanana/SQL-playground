# HW5 — Поиск в массивах, CTE, подзапросы и материализованные представления

## Цели домашнего задания

- закрепить навыки использования функций и операторов языка SQL по поиску значений в массиве;
- продемонстрировать работу с материализованными представлениями;
- научиться строить и анализировать план выполнения запросов.

---

Решения: [`HW5_Solutions.sql`](./HW5_Solutions.sql)

---

## Задания:

### Задание 1

Написать SQL-запрос, который выводит всю информацию о фильмах со специальным атрибутом `"Behind the Scenes"`.

**Требования к результату:**

- Название фильма;
- Столбец со специальными атрибутами.

---

### Задание 2

Написать ещё **2 варианта** поиска фильмов с атрибутом `"Behind the Scenes"`, используя **другие функции или операторы SQL** для поиска значения в массиве.

**Требования к результату:**

- Название фильма;
- Столбец со специальными атрибутами.

---

### Задание 3

Для каждого покупателя посчитать, сколько он брал в аренду фильмов со специальным атрибутом `"Behind the Scenes"`.

**Обязательное условие:**

- использовать запрос из задания 1, помещённый в **CTE**.

**Требования к результату:**

- Фамилия и имя пользователя в виде одного значения;
- Количество арендованных фильмов.

---

### Задание 4

Для каждого покупателя посчитать, сколько он брал в аренду фильмов со специальным атрибутом `"Behind the Scenes"`.

**Обязательное условие:**

- использовать запрос из задания 1, помещённый в **подзапрос**, который необходимо использовать для решения задания.

**Требования к результату:**

- Фамилия и имя пользователя в виде одного значения;
- Количество арендованных фильмов.

---

### Задание 5

Создать **материализованное представление** с запросом из предыдущего задания и написать запрос для **обновления** материализованного представления.

---

### Задание 6

С помощью `EXPLAIN ANALYZE` провести анализ стоимости выполнения запросов из предыдущих заданий и ответить на вопросы:

- с каким оператором или функцией SQL, использованными при выполнении домашнего задания, поиск значения в массиве затрачивает **меньше ресурсов системы**;
- какой вариант вычислений затрачивает меньше ресурсов системы: с использованием **CTE** или с использованием **подзапроса**.

## Дополнительная часть

### Задание 1

1. Открыть по [ссылке](https://letsdocode.ru/sql-main/sql-hw5.sql) SQL-запрос.  
2. Сделать `EXPLAIN ANALYZE` этого запроса.  
3. Основываясь на описании запроса, найти **узкие места** и описать их.  
4. Сравнить с вашим запросом из основной части (если ваш запрос изначально укладывается в 15 мс — отлично!).  
5. Сделать **построчное описание** `EXPLAIN ANALYZE` на русском языке для оптимизированного запроса.  
   - Описание строк в `EXPLAIN` можно посмотреть по [ссылке](https://use-the-index-luke.com/sql/explain-plan/postgresql/operations).

### Задание 2

Используя оконную функцию, вывести для каждого сотрудника сведения о **первой его продаже**.

**Требования к результату:**

- Все столбцы из таблицы с платежами.

---

### Задание 3

Для каждого магазина определить и вывести одним SQL-запросом следующие аналитические показатели:

- день, в который арендовали **больше всего фильмов** (в формате `год-месяц-день`);
- количество фильмов, взятых в аренду в этот день;
- день, в который продали фильмов на **наименьшую сумму** (в формате `год-месяц-день`);
- сумму продажи в этот день.

**Требования к результату:**

- Идентификатор магазина;
- День аренды;
- Количество фильмов;
- День продажи;
- Сумма продаж.
