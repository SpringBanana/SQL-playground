# HW4 — Оконные функции и подзапросы

## Цель домашнего задания

- закрепить навыки работы с оконными функциями, табличными выражениями и подзапросами.

---

Решения: [`HW4_Solutions.sql`](./HW4_Solutions.sql)

---

## Задания:

### Задание 1

Сделать запрос к таблице `payment` и с помощью оконных функций добавить вычисляемые колонки согласно условиям:

1.1 Пронумеровать все платежи от 1 до N по дате платежа.  
1.2 Пронумеровать платежи для каждого покупателя, сортировка платежей — по дате платежа.  
1.3 Посчитать нарастающим итогом сумму всех платежей для каждого покупателя, сортировка — по дате платежа, а затем по размеру платежа от наименьшего к большему.  
1.4 Пронумеровать платежи для каждого покупателя по размеру платежа от наибольшего к меньшему так, чтобы платежи с одинаковым значением имели одинаковый номер.

**Требования к результату:**

- Идентификатор платежа;
- Дата платежа;
- Идентификатор пользователя;
- Размер платежа;
- 4 столбца с результатами оконных функций.

---

### Задание 2

С помощью оконной функции вывести для каждого покупателя стоимость платежа и стоимость платежа **из предыдущей строки** со значением по умолчанию `0.0` при сортировке по дате платежа.

**Требования к результату:**

- Идентификатор платежа;
- Дата платежа;
- Идентификатор пользователя;
- Текущий размер платежа;
- Размер платежа из предыдущей строки.

---

### Задание 3

С помощью оконной функции определить, на сколько каждый следующий платеж покупателя больше или меньше текущего.

**Требования к результату:**

- Идентификатор платежа;
- Дата платежа;
- Идентификатор пользователя;
- Текущий размер платежа;
- Следующий размер платежа;
- Разница между текущим и следующим платежами.

---

### Задание 4

С помощью оконной функции для каждого покупателя вывести данные о его **последней оплате аренды**.

**Требования к результату:**

- Все столбцы из таблицы с платежами.

## Дополнительная часть

### Задание 1

С помощью оконной функции вывести для каждого сотрудника сумму продаж за **август 2005 года** с нарастающим итогом по каждому сотруднику и по каждой дате продажи (без учёта времени) с сортировкой по дате.

**Требования к результату:**

- Фамилия и имя сотрудника в виде одного значения;
- Сумма продаж на каждый день;
- Накопительный итог.

---

### Задание 2

20 августа 2005 года в магазинах проходила акция: покупатель **каждого сотого платежа** получал дополнительную скидку на следующую аренду.  
С помощью оконной функции вывести всех покупателей, которые в день проведения акции получили скидку.

**Требования к результату:**

- Идентификатор пользователя;
- Фамилия и имя пользователя в виде одного значения.

---

### Задание 3

Для каждой страны определить и вывести одним SQL-запросом покупателей, которые попадают под условия:

- покупатель, арендовавший **наибольшее количество фильмов**;
- покупатель, арендовавший фильмы на **самую большую сумму**;
- покупатель, который **последним арендовал фильм**.

**Требования к результату:**

- Название страны;
- Фамилия и имя пользователя в виде одного значения — лучшего по количеству;
- Фамилия и имя пользователя в виде одного значения — лучшего по сумме платежей;
- Фамилия и имя пользователя в виде одного значения — последним арендовавшего фильм.

Дополнительно:

- можно реализовать вариант, когда при наличии нескольких пользователей в топ‑1 выбирается **один случайный**;
- либо вариант, когда выводятся **все пользователи**, попавшие в топ‑1.  
Выбор варианта остаётся за вами.
